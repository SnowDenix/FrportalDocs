## Purpose

This document establishes standardized procedures for stock transfers between franchise stores. Effective transfer management optimizes regional inventory, improves product availability, and enhances overall sales performance.

### Key Objectives

| Objective | Impact |
|-----------|--------|
| **Balance Inventory** | Optimize regional stock levels |
| **Reduce Markdowns** | Move slow sellers to better locations |
| **Improve Availability** | Fill gaps in fast-selling items |
| **Minimize Losses** | Accurate transfer tracking |
| **Enhance Sales** | Right product in right location |

## Scope

### Transfer Types

| Transfer Type | Description | Approval |
|---------------|-------------|----------|
| **Store-to-Store** | Direct transfer between stores | Regional Manager |
| **Store-to-DC** | Return to distribution center | HQ Approval |
| **DC-to-Store** | Emergency replenishment | HQ Initiated |
| **Regional Rebalance** | Multiple store adjustments | Regional Manager |
| **Season End** | End-of-season consolidation | HQ Approval |

### Transfer Reasons

| Reason | Priority | Typical Volume |
|--------|----------|----------------|
| **Stock-out Prevention** | High | 10-50 units |
| **Size/Color Rebalance** | Medium | 20-100 units |
| **Slow-Mover Relocation** | Medium | 50-200 units |
| **Store Opening Support** | High | Varies |
| **Store Closing Transfer** | High | Full inventory |
| **Event/Promotion Support** | High | As needed |

### Transfer Request Flow

```
TRANSFER REQUEST WORKFLOW

1. INITIATION
   └── Receiving store identifies need
       └── Checks regional inventory
           └── Submits request

2. APPROVAL ROUTING
   └── Request → Regional Manager
       └── Review within 24-48 hours
           ├── APPROVED → Proceed to Step 3
           └── REJECTED → End (with reason)

3. EXECUTION
   └── Sending store notified
       └── Pick and pack (24 hours)
           └── Ship/deliver

4. COMPLETION
   └── Receiving store receives
       └── Verifies and confirms
           └── Transfer closes
```

## Examples / Scenarios

### Scenario 1: Urgent Stock-out Prevention

**Situation:** Store A running low on best-seller, Store B has excess

**Process:**
```
Day 1 - Morning:
1. Store A identifies stock-out risk
2. Checks regional inventory system
3. Sees Store B has 30 units excess
4. Submits urgent transfer request (15 units)

Day 1 - Afternoon:
5. Regional Manager approves (urgent)
6. Store B picks and packs items
7. Store B updates inventory (sends out)
8. Arranges same-day courier

Day 2 - Morning:
9. Store A receives transfer
10. Verifies 15 units received
11. Updates inventory (receives in)
12. Confirms transfer complete
```

**Result:** Stock-out prevented, sales maintained

---

### Scenario 2: Size Rebalancing

**Situation:** Store A has excess Size M, Store B needs Size M

**Process:**
```
1. Regional Manager reviews size distribution
2. Identifies imbalance between stores
3. Creates rebalancing transfer order
4. Store A prepares size M items
5. Store A ships to Store B
6. Store B receives and processes
7. Both stores update inventory
```

**Result:** Size availability optimized across region

---

### Scenario 3: Transfer with Discrepancy

**Situation:** Receiving store gets fewer items than expected

**Process:**
```
1. Transfer of 50 units sent
2. Receiving store counts 47 units
3. Discrepancy documented immediately
4. Photos taken of received items
5. Sending store contacted
6. Sending store checks their records
7. Investigation determines 3 items misplaced
8. Additional 3 units sent
9. Transfer completed
10. Incident logged for review
```

**Result:** Discrepancy resolved, process improved

---

### Scenario 4: Season-End Consolidation

**Situation:** End of season, consolidate remaining stock

**Process:**
```
1. HQ identifies consolidation stores
2. Transfer orders created for all stores
3. Multiple stores send to consolidation point
4. Items received and processed
5. Inventory updated at all locations
6. Consolidated stock prepared for clearance
```

**Result:** Efficient season-end inventory management

## Common Issues & Solutions

### Issue 1: Delayed Approvals

**Problem:** Transfer requests pending too long

**Solutions:**
- Set clear escalation timelines
- Enable auto-approval for small transfers
- Use mobile approval for Regional Managers
- Follow up on pending requests
- Review approval process efficiency

---

### Issue 2: Transfer Variances

**Problem:** Items missing or damaged in transit

**Solutions:**
- Double-check packing before shipment
- Use secure packaging
- Take photos at send and receive
- Document all items carefully
- Choose reliable carriers

---

### Issue 3: System Update Delays

**Problem:** Inventory not updated timely

**Solutions:**
- Update system immediately
- Train staff on process
- Set reminders for updates
- Verify updates complete
- Audit transfer records

---

### Issue 4: Rejection of Good Requests

**Problem:** Valid transfer requests rejected

**Solutions:**
- Provide clear justification
- Include supporting data (sales velocity)
- Communicate urgency clearly
- Follow up with Regional Manager
- Request reconsideration with additional info
