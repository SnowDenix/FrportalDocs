## Purpose

This document establishes standardized procedures for store opening and closing operations across all Aydınlı Group franchise locations. Consistent execution of these procedures ensures security, operational readiness, and compliance with brand standards.

### Objectives

- **Security:** Protect assets, staff, and premises
- **Consistency:** Standardized process across all locations
- **Readiness:** Store prepared for optimal customer experience
- **Accountability:** Clear responsibilities and documentation
- **Compliance:** Adherence to legal and brand requirements

---

## Scope

### Application

These procedures apply to:
- All store formats (Flagship, Standard, Compact, Shop-in-Shop, Outlet)
- All geographic regions
- All operating days including special trading hours

### Operating Hours

| Day | Standard Hours | Mall Hours | Holiday Hours |
|-----|----------------|------------|---------------|
| Monday-Friday | 10:00-21:00 | Per mall | Variable |
| Saturday | 10:00-22:00 | Per mall | Variable |
| Sunday | 11:00-20:00 | Per mall | Variable |

*Note: Hours vary by location and local regulations*

## Definitions

| Term (EN) | Term (TR) | Definition |
|-----------|-----------|------------|
| Opening Float | Açılış Nakiti | Predetermined cash amount to start the day |
| Z-Report | Z Raporu | End-of-day fiscal register report |
| X-Report | X Raporu | Mid-day register reading (non-resetting) |
| Safe Drop | Kasaya Bırakma | Transfer of excess cash to safe |
| Key Holder | Anahtar Sorumlusu | Staff authorized to open/close store |
| Alarm Code | Alarm Kodu | Security system deactivation code |
| Pre-Opening | Açılış Öncesi | Time period before store opens to customers |
| Post-Closing | Kapanış Sonrası | Time period after last customer leaves |

## Examples / Scenarios

### Scenario 1: Alarm Triggers During Opening

**Situation:** Alarm sounds when entering store

**Response Protocol:**
1. Do NOT proceed into store
2. Exit immediately to safe distance
3. Call alarm monitoring company
4. Call police if advised
5. Call Store Manager / Regional Manager
6. Do not re-enter until cleared
7. Document incident fully

### Scenario 2: Customer Refuses to Leave at Closing

**Situation:** Customer still shopping 10 minutes past close

**Response Protocol:**
1. Politely approach customer
2. "I'm sorry, we've now closed. May I help you complete your purchase?"
3. Offer to hold items if needed
4. Escort to checkout
5. If refuses, involve manager
6. Never physically force customer
7. Document if significant delay

### Scenario 3: Cash Variance at Closing

**Situation:** $15 short in cash drawer

**Response Protocol:**
1. Re-count entire drawer
2. Check for miscounted bills
3. Review void/return transactions
4. Check for bills stuck together
5. Document variance on form
6. Report to Manager immediately
7. Complete incident report
8. Do not adjust without authorization

## Common Issues & Solutions

### Issue 1: Alarm Won't Deactivate

**Solution:**
1. Try code again carefully
2. Check for correct code
3. Call alarm company immediately
4. Have ID ready for verification
5. Do not disable manually

### Issue 2: POS System Won't Start

**Solution:**
1. Check power connections
2. Try restart
3. Call IT support
4. Use backup procedures
5. Manual transaction recording if needed

### Issue 3: Cash Float Incorrect

**Solution:**
1. Re-count with witness
2. Check previous day's closing
3. Contact manager
4. Document discrepancy
5. Do not adjust without approval
