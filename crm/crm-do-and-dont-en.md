## Purpose

This document defines the mandatory rules and absolutely prohibited actions for CRM usage at Aydınlı Group franchise stores (U.S. Polo Assn., Pierre Cardin, Cacharel).

**This document is mandatory for all store staff.**

## General Rules

### ✅ DO's

| # | Rule | Why Important |
|---|------|---------------|
| 1 | Match customer at every sale | Critical for KPIs and customer tracking |
| 2 | Enter data correctly the first time | Correction is costly and time-consuming |
| 3 | Always obtain GDPR consents | Legal requirement |
| 4 | Report suspicious situations immediately | Necessary for early intervention |
| 5 | Log out of system securely | Essential for data security |
| 6 | Record customer complaints | Important for service quality |
| 7 | Know current campaigns | Required for customer satisfaction |
| 8 | Attend trainings | Mandatory for competency |

### ❌ DON'Ts

| # | Rule | Consequence |
|---|------|-------------|
| 1 | Don't process data without consent | GDPR penalty |
| 2 | Don't enter fake data | Data quality degrades |
| 3 | Don't share your password | Security breach |
| 4 | Don't export customer data | Legal sanction |
| 5 | Don't access unauthorized screens | Disciplinary action |
| 6 | Don't force customers to register | Customer loss |
| 7 | Don't leave tickets unanswered | SLA violation |
| 8 | Don't apply off-system discounts | Financial loss |

## Customer Registration

### ✅ DO

| Rule | Explanation |
|------|-------------|
| ✅ Search for existing customer before registration | To prevent duplicates |
| ✅ Verify phone number | To prevent incorrect data |
| ✅ Confirm email letter by letter | To prevent bounces |
| ✅ Register with customer's consent | For GDPR compliance |
| ✅ Leave blank if customer won't provide info | Instead of fake data |
| ✅ Tell customer about welcome bonus | For satisfaction |

### ❌ DON'T

| Rule | Risk |
|------|------|
| ❌ Don't directly create new registration | Creates duplicates |
| ❌ Don't guess phone numbers | Wrong person contacted |
| ❌ Don't enter fake emails like aaa@aaa.com | Data quality drops |
| ❌ Don't enter made-up birth dates | Wrong campaign sent |
| ❌ Don't register without consent | GDPR violation |
| ❌ Don't force customers to register | You'll lose customers |

## Data Security and GDPR

### ✅ DO

| Rule | Explanation |
|------|-------------|
| ✅ Always show/read the privacy notice | GDPR requirement |
| ✅ Obtain data processing consent | Required for registration |
| ✅ Ask communication permissions separately | Separate consent for each channel |
| ✅ Archive consent forms | Required for audits |
| ✅ Lock your screen when leaving desk | Security |
| ✅ Change your password regularly | Security |
| ✅ Report suspicious access | Security |

### ❌ DON'T

| Rule | Consequence |
|------|-------------|
| ❌ Don't process data without consent | Significant fine risk |
| ❌ Don't ask for sensitive data (health, religion, etc.) | Legal sanction |
| ❌ Don't share customer data | Data breach |
| ❌ Don't export data via USB/email | Serious sanction |
| ❌ Don't share your password with anyone | Security breach |
| ❌ Don't use someone else's account | Audit difficulty |
| ❌ Don't write customer info on paper | Data security risk |

## Important Warnings

---

## Violation Consequences

### Violation Levels

| Level | Example | Consequence |
|-------|---------|-------------|
| **Low** | Data format error | Warning + correction |
| **Medium** | Repeated negligence | Written warning |
| **High** | GDPR violation, data sharing | Disciplinary action |
| **Critical** | Intentional abuse | Contract termination + legal action |

### Violation Reporting

When violation detected:
1. Immediately inform store manager
2. Preserve evidence
3. Store manager reports to regional manager
4. CRM Headquarters is informed
